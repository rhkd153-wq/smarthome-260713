# 스마트 홈 솔루션 (Smart Home Solution)

신체장애인의 자립과 삶의 질 향상을 위한 **스마트 홈 가정 환경 수정 평가 도구**입니다.
평가자(작업치료사 등)는 누구나 정해진 순서대로 진행하여 대상자에게 맞는 스마트 홈 솔루션을 도출할 수 있습니다.

> 근거: 문광태 외, *스마트 홈 가정 환경 수정 프로그램 매뉴얼* (보건복지부, 2023) 및 강의록
> — 양식지 1·2·3 (부록 4·5·6)과 적용 알고리즘을 기반으로 구성했습니다.

## 평가 흐름

강의록의 처리 과정(**평가 → 중재 → 결과**)과 적용 알고리즘을 따릅니다.

| 단계 | 내용 | 근거 양식지 |
|------|------|-------------|
| **1. 기초 스크리닝** | **현재 활동 수준**을 먼저 확인한 뒤, 개인 능력·환경 평가 | 양식지 1 (부록 4) |
| **2. 공간별 수행도** | 현관·침실·거실/부엌·화장실의 작업별 수행 정도(1·2·3) 평가 | 양식지 2 (부록 5) |
| **3. 솔루션 제안** | 제한되는 활동(수행도 1·2)에 맞는 적정 스마트 홈 기술 자동 도출 | 양식지 3 (부록 6) |

- **수행도 척도**: 미경험 · 1(못함/안함) · 2(도움받아 함) · 3(독립적으로 수행함)
- 수행도 **1 또는 2**로 평가된 활동이 솔루션(적정 스마트 홈 기술) 추천 대상이 됩니다.
- 결과 화면에서 **인쇄 / PDF 저장**으로 솔루션 제안서를 출력할 수 있습니다.


## 주요 기능 (v2)

- **개인·환경 반영 필터**: 손의 기능적 사용·대화 가능 여부·OS·Wi-Fi 유무를 반영해 **권장 제어 인터페이스**(스마트폰 앱/리모컨·큰 스위치·음성·자동화)와 권장 플랫폼을 제안하고, Wi-Fi 미보유 시 해당 기기를 "구축 필요"로 표시합니다.
- **결과 저장·불러오기**: 평가 내용이 브라우저에 자동 저장되어 새로고침·재방문 시 이어서 진행할 수 있고, JSON 파일로 **내보내기/불러오기** 하여 여러 대상자를 관리할 수 있습니다.
- **초보자/숙련자 키트 + 통합 기기 목록**: 도출된 기기를 설치 난이도에 따라 초보자(플러그앤플레이)·숙련자(허브·자동화) 키트로 묶어 제시하고, 중복을 제거한 통합 구매/설치 목록과 각 기기가 지원하는 활동을 보여줍니다. IoT 사용 경험으로 등급을 자동 판정하며 수동 변경도 가능합니다.
- **제안서 서식**: 대상자 정보표 · 권장 제어 인터페이스 · 공간별 적정 기술 · 중재 계획(목표·예산·자동화 시나리오) · A/S 안내를 포함하며 인쇄/PDF로 출력됩니다.

## 실행 방법

별도 설치나 빌드 없이 동작하는 정적 웹앱입니다.

- **바로 열기**: `index.html` 파일을 웹 브라우저로 엽니다.
- **로컬 서버(권장)**:
  ```bash
  python3 -m http.server 8000
  # 브라우저에서 http://localhost:8000 접속
  ```
- **웹 배포**: GitHub Pages, Netlify 등 정적 호스팅에 그대로 올리면 됩니다.

## Netlify 배포 (누구나 사용할 수 있는 공개 URL)

빌드가 필요 없는 정적 사이트라, 아래 두 방법 중 하나로 몇 분 만에 공개할 수 있습니다.

### 방법 A. 드래그 앤 드롭 (가장 빠름, GitHub 불필요)
1. https://app.netlify.com/drop 접속 (Netlify 로그인)
2. 이 저장소 폴더(또는 배포용 zip의 압축을 푼 폴더)를 페이지에 끌어다 놓기
3. 자동으로 `https://<임의이름>.netlify.app` 공개 URL 생성 → 완료

### 방법 B. GitHub 연동 (변경 시 자동 재배포, 권장)
1. https://app.netlify.com → **Add new site → Import an existing project**
2. GitHub 연결 후 이 저장소(`smarthome-260713`) 선택
3. 설정값:
   - **Branch to deploy**: 배포할 브랜치 선택
   - **Build command**: 비워 둠 (없음)
   - **Publish directory**: `.` (루트) — `netlify.toml`에 이미 설정됨
4. **Deploy** → 공개 URL 생성. 이후 저장소에 푸시하면 자동 재배포됩니다.

> 사이트 이름(`***.netlify.app`)은 Netlify 사이트 설정의 **Site name**에서 원하는 이름으로 바꿀 수 있습니다.

## 파일 구조

```
index.html          진입 화면
assets/
  data.js           도메인 데이터 (공간→작업→활동→적정 기술) · 양식지 내용
  app.js            화면 흐름 및 로직
  styles.css        스타일
```

양식지 내용(활동·적정 기술 목록)을 수정하려면 `assets/data.js` 하나만 편집하면 됩니다.
